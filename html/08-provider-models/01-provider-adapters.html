01 · Provider 适配层

学习要点

  • Provider 适配层在架构中的定位是什么？Agent 调用和底层 Provider 如何解耦？
  • OpenClaw 支持哪些 Provider？各自的认证方式和特点是什么？
  • 模型选择流程考虑哪些因素？primary + fallbacks 的配置模式如何工作？
  • Provider 适配层的三层架构（调用层 → 适配层 → 实现层）如何协同？

1. Provider 层概述

AI Provider 层是 Gateway 的模型中枢，负责对接各种 LLM 提供商，提供统一的模型调用接口。

3. 适配架构

Provider 适配层采用三层架构，上层（调用层）无需关心底层使用哪个 Provider：

模型命名规范

OpenClaw 使用 provider/model 格式唯一标识一个模型：

{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",   // 主模型
        fallbacks: ["openai/gpt-5.2"],             // 回退模型
      },
    },
  },
}
命名示例 对应的 Provider 模型
anthropic/claude-sonnet-4-5 Anthropic Claude Sonnet 4.5
openai/gpt-5.2 OpenAI GPT 5.2
ollama/llama3 Ollama Llama 3（本地）
gemini/gemini-2.0-flash Google Gemini 2.0 Flash

4. 模型选择流程

当 Agent 发起模型调用时，按以下流程选择模型：

flowchart LR REQ["Agent 请求"]:::start --> FACTORS{"选择因素"}:::factors FACTORS --> F1["模型能力 工具调用 / 视觉 / 语音"]:::factor FACTORS --> F2["上下文窗口 Token 上限"]:::factor FACTORS --> F3["成本考虑 API 费用 / 预算"]:::factor FACTORS --> F4["可用性 服务状态"]:::factor F1 & F2 & F3 & F4 --> PRIMARY["选择主模型 primary"]:::result PRIMARY --> FB["配置回退列表 fallbacks"]:::result classDef start fill:#f1f5f9,stroke:#64748b,color:#1e293b classDef factors fill:#fef3c7,stroke:#f59e0b,color:#92400e classDef factor fill:#dbeafe,stroke:#3b82f6,color:#1e40af classDef result fill:#d1fae5,stroke:#10b981,color:#065f46
选择因素 说明 影响
模型能力 是否需要工具调用、视觉识别、语音处理 决定可选模型范围
上下文窗口 模型支持的最大 Token 数（4K / 8K / 32K / 100K / 200K） 决定能处理多长的对话
成本考虑 API 调用费用，不同模型价格差异大 影响日常运行成本
可用性 模型服务是否在线、有无速率限制 影响可靠性

5. 模型配置示例

{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-5",
        fallbacks: ["openai/gpt-5.2"],
      },
      // 模型别名（可选）
      models: {
        "anthropic/claude-sonnet-4-5": { alias: "Sonnet" },
        "openai/gpt-5.2": { alias: "GPT" },
      },
    },
  },
}

CLI 快速查看

# 列出可用模型
openclaw models list

# 查看当前配置
openclaw config get agents.defaults.model
