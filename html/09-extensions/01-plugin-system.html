01 · 插件系统

学习要点

  • 插件系统的四层架构（manifest → 规划 → 注册 → 查找）分别做什么？
  • 插件支持哪五种注入点？各自的用途是什么？
  • 插件的完整生命周期（开发 → 加载 → 运行时）是怎样的？
  • 如何获取和安装插件？插件目录的扫描顺序是怎样的？

1. 插件系统概述

插件是 OpenClaw 的扩展骨架，通过四层架构实现功能扩展。一个插件可以同时提供新通道、新工具、新 Provider、新钩子和新 HTTP 路由。

flowchart LR subgraph 四层架构["插件系统四层"] direction LR L1["📄 manifest 元数据声明"]:::m1 L2["📋 规划 启用 / 加载"]:::m2 L3["⚡ runtime register 运行时注册"]:::m3 L4["🔍 registry 注册表查找"]:::m4 end L1 -->|"声明能力"| L2 L2 -->|"实例化"| L3 L3 -->|"注册到注册表"| L4 classDef m1 fill:#dbeafe,stroke:#3b82f6,color:#1e40af classDef m2 fill:#fef3c7,stroke:#f59e0b,color:#92400e classDef m3 fill:#d1fae5,stroke:#10b981,color:#065f46 classDef m4 fill:#fce7f3,stroke:#ec4899,color:#9d174d
层 名称 职责
① manifest 插件元数据声明：名称、版本、入口、注入点、依赖
② 规划 插件启用决策、依赖解析、加载顺序规划
③ 运行时注册 将插件功能注册到系统注册表
④ registry 运行时按类型（channel/tool/hook 等）查找已注册的功能

2. 五种注入点

插件通过声明 injects 字段指定要扩展的能力类型：

注入点 扩展什么 示例
channel 新消息通道 Telegram、Discord、Slack 适配器
tool 新 Agent 工具 自定义脚本执行器、API 集成工具
provider 新 AI Provider Ollama、Azure OpenAI、自定义模型
hook 生命周期钩子 message_received、gateway_stop
http-route 新 HTTP 路由 Webhook 接收、自定义 API 端点

获取插件的方式

方式 说明 命令
ClawHub 官方插件市场 openclaw plugins install <name>
本地安装 从本地目录加载 放在 ~/.openclaw/plugins/ 下
工作区安装 工作区专用插件 放在 workspace/plugins/ 下

4. manifest 元数据

{
  name: "my-plugin",           // 插件唯一标识
  version: "1.0.0",            // 语义化版本
  description: "自定义工具插件", // 描述
  entry: "./dist/index.js",    // 入口文件
  injects: ["channel", "tool"], // 注入点类型
  dependencies: {
    // 依赖的其他插件
    "base-plugin": "^1.0.0",
  },
}
字段 是否必需 说明
name ✅ 插件唯一标识，全局唯一
version ✅ 语义化版本号
description ❌ 插件描述
entry ✅ 入口 JS 文件路径
injects ✅ 注入点类型数组
dependencies ❌ 依赖的其他插件（版本范围）

文件 作用
